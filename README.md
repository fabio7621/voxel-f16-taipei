# VOXEL SQUADRON — F-16 OVER TAIPEI

一個純前端的 3D 像素風(Voxel)互動飛行展示:三架點陣 F-16 編隊繞行台北市上空,可以點擊任一架戰機接管駕駛、低空穿越台北 101、發射飛彈。整個專案只有一個 `index.html`,雙擊開啟即可遊玩,無需安裝或建置。

## 操作方式

### 環繞模式(預設)
| 操作 | 功能 |
|------|------|
| 拖曳 | 360° 環繞檢視編隊 |
| 滾輪 / 雙指縮放 | 調整鏡頭距離 |
| 點擊戰機 | 接管該架戰機駕駛 |

### 駕駛模式
| 操作 | 功能 |
|------|------|
| `↑` `↓` / `W` `S` | 俯仰(Pitch) |
| `←` `→` / `A` `D` | 滾轉(Roll) |
| `Shift` | 加力(加速) |
| `空白鍵` / 點擊畫面 | 發射飛彈 |
| `Esc` / 「返回編隊」按鈕 | 交還控制,戰機自動歸隊 |
| 虛擬搖桿(左下角) | 觸控裝置的俯仰 / 滾轉操作 |

## 使用技術

### 核心
- **HTML5 + CSS3 + Vanilla JavaScript(ES6+)** — 單檔架構,零建置工具、零框架依賴
- **[Three.js r128](https://threejs.org/)**(經 cdnjs CDN 載入)— 3D 場景渲染引擎,使用 WebGL

### Three.js 渲染技術
- **WebGLRenderer 低解析度渲染**:以 0.5 倍解析度渲染再用 CSS `image-rendering: pixelated` 放大,營造復古點陣畫面並提升效能
- **InstancedMesh 實例化渲染**:街區建築(依色票分桶)、行道樹、屋頂水塔皆以 InstancedMesh 繪製,大幅減少 draw call
- **共用幾何體與材質快取**:全場景共用單一 `BoxGeometry`,材質依顏色快取重用
- **MeshLambertMaterial + HemisphereLight / DirectionalLight**:低成本的漫反射光照
- **AdditiveBlending 粒子特效**:飛彈尾煙、爆炸火光、引擎噴流
- **Raycaster 點擊偵測**:點選戰機接管駕駛

### 飛行模擬(簡化空氣動力學)
- **協調轉彎**:轉彎率 ω = g·tanφ / V,必須壓坡度(bank)才會轉彎,速度越快轉彎半徑越大
- **能量管理**:爬升損失空速、俯衝換取空速(g·sinγ 沿航跡分量)
- **失速模擬**:空速低於失速速度時機鼻自動下垂,並顯示 STALL 警告
- **升力分量**:大坡度轉彎時垂直升力減少,會自然掉高度
- **第三人稱追蹤鏡頭**:以指數平滑(exponential lerp)跟隨戰機

### 輸入與互動
- **Pointer Events API**:統一處理滑鼠與觸控(拖曳環繞、雙指捏合縮放、點擊射擊)
- **自製虛擬搖桿**:行動裝置上的觸控飛行操作
- **鍵盤事件**:WASD / 方向鍵飛行控制

### 場景內容(程序化點陣台北)
- 台北 101(八節斗形樓段、如意金飾、塔頂警示燈閃爍)
- 台北大巨蛋、中正紀念堂、圓山大飯店、大安森林公園
- 蜿蜒的淡水河與跨河橋、棋盤式街道網格、隨機生成的街區建築與車流

## 執行方式

```bash
# 直接用瀏覽器開啟
open index.html

# 或起一個本機伺服器
python3 -m http.server 8000
# 瀏覽 http://localhost:8000
```

> 需要網路連線載入 Three.js CDN;支援桌面與行動裝置瀏覽器(需支援 WebGL)。
